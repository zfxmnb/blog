      jquery事件绑定

      Post on 2016-07-14

      jquery的版本更新历程中，有很多方法发生了改变，其中包括我们最常见的绑定事件方法。说到绑定事件方法就不得不提到jquery的两个版本，1.70版本和1.90版本，在这两个版本中jquery的绑定事件方法发生了一下改变。

      在1.70版本中首次引进了on方法，on方法是所有事件绑定方法中最强大的。至于他为什么是最强大的呢，后面会详细讲到。之前说到另一个jquery版本1.90，这个版本把live方法从绑定事件方法家族中驱逐出去，这是在jquery以往的升级中很少出现的这种把一个已有的方法去掉的情况，因为这样会导致升级后项目出现错误。其实除了这两种方法外还有几种事件绑定方法。它们分别是bind、delegate、one.

      那现在就来看一下上面这五种方法有哪些相似和差异。


          

      <div class="div2">
</div><br>

<div class="div3">
    <button class="delegate">delegate点击按钮</button>
</div><br>

<div class="div4">
    <button class="bind">bind点击按钮</button>
    <button class="test">test点击按钮</button>
</div><br>

<div class="div5">
    <button class="one">one点击按钮</button>
</div></pre>


       //动态添加on和off按钮
          $(“.div3”).click(function(){
              $(“.div1”).html(‘‘);
              $(“.div2”).html(‘‘)
          });

      //on事件绑定方法
$(".div1").on("click",".on",data1="zfx1",function(){
     console.log("this:"+$(this).attr("class"));    //this:on
    console.log("data:"+data1);                     //data:zfx1
});

//live事件绑定方法
$(".div2").live("click",".live",function(){
     console.log("this:"+$(this).attr("class"));    //this:div2
 });
$(".div2 .test").live("click",".test",function(){
     console.log("this is test");                   //this is test
                                                    //this:div2
 });

//delefate
$(".div3").delegate(".delegate","click",data2="zfx3",function(){
    console.log("this:"+$(this).attr("class"));     //this:delegate
    console.log("data:"+data2);                     //data:zfx3
});

//bind事件绑定方法
$(".div4").bind("click",".bind",function(){
    console.log("this:"+$(this).attr("class"));     //this:div4
});
$(".div4 .test").click(function(){
     console.log("this is test");                   //this is test
                                                    //this:div4
 });

//one单次事件绑定方法
$(".div5").one("click",".one",data3="zfx5",function(){
     console.log("this:"+$(this).attr("class"));    //this:one
    console.log("data:"+data3);                     //data:zfx5
 });</pre>

      总结上面代码的运行结果可以把这些绑定事件方法特点总结一下。

      on方法是比较最新的事件绑定方法，它结合了其他几种方法的优点同时也抛掉了其他几种方法的一些缺点，比如，它有live方法的元素预绑定功能又可以像delegate方法一样向函数里面传入值，虽然这个功能用的比较少，但是好像确实有点鸡肋，同时在函数通过$(this)获取的当前对象是方法选择的对象，而不是选择器对象。

      live方法最主要的功能是预绑定，这个方法在jquery1.9.0之前可以使用的，之后就由on方法代替，这样导致很多项目不能升级到最新的jquery版本，那为什么还要坚持去除这一方法呢，这不得不说live的另一个特点，它绑定事件的方法类似bind,bind方法对一个对象绑定事件是对选择的对象及其子元素分别绑定事件，live也是一样，而预绑定会占用浏览器性能，这就导致了它的预绑定的性能远远不如on方法。

      delegate方法则比较像on方法，除了不能像on一样预绑定事件外，其他特点几乎和on一样。

      bind方法前面已经说了它的一个最重要的特点就是对选择的对象及其子元素分别绑定事件，这样的功能其实在很多时候还是可以用上的，它的另一个特点就是不能向函数里面传入参数，但个人觉得这无关紧要。

      最后要说的是单次绑定事件方法one方法，这个方法可以只触发一次事件，这个方法的运用场景非常多，这里就不多做介绍，这个方法除了这个特点就是它的其他功能和on方法一模一样。

      除了这些绑定事件的方法还有对应绑定事件对应的解绑事件方法，作用都是对已绑定的事件的对象清除绑定，让事件失效，具体的有哪些解绑方法和它们的对应关系这里就不多做详细说明。
