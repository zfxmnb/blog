      web上传图片原理

      Post on 2016-07-20

      说到网页中上传图片，通常会使用想到表单input[type=file]的方式上次图片，但是有多少人并不知道在这个过程中浏览器和服务器对图片数据是怎样处理的，在这篇文章里我将对上传图片的过程进行一些简单的描述，有什么不对的地方还望指教。

      我们通常用<input type="file">在网页上打开本地文件预览，然后选着上传文件，然后通过表单提交吧图片信息提交到服务器，异或者是通过Ajax或者iframe提交数据，后者可以不用刷新页面也可以提交数据，前面说到这样的方式是提交文件信息其中包括文件的路径、文件名、文件类型、文件大小等关键信息，其中不包括文件的数据，服务器要获取文件数据需要通过路径去读取，这个读取的过程就是文件上传的实现，这个上传的全过程不是一次性提交完成的，而是提交一个过程，读取一个过程，这个全过程是一个典型的文件上传过程，但这并不是现在我要说的图片上传方式，把打开的图片转化为可传输的base64数据格式，在通过表单或者ajax或者通过iframe上传，传输方式通常选择post,因为post要上传的数据要比get所能上传数据要大得多；而把数据传输到服务器端后，服务器就把接收到的base64数据通过处理后已特定的名称保存在特定的文件夹下，这就是另一种上传图片的的全过程，这种上传图片的方法相较与典型的上传方法要更直接些，但这不能说它就比前者要更有优势，因为这种方式上次的只是图片的基本数据，没有过多的诸如文件名文件大小等信息，但现在HTML5越来越普及，浏览器器可调用的资源更多了，其中与图像相关的有canvas绘图、摄像头拍摄，canvas绘制的图像和摄像头拍摄图片需要上传到服务器这不能用之前的典型上传方式，因为它们都是没有路径的，只能通过base64数据格式的上传，说了这么多base64数据格式的图片上传下面来说一下关键性的代码。

      首先来说一下前端的实现，前端部分包括图片数据的处理和上传数据两个步。

      第一步，把图片数据处理成base64数据格式，这里处理相对较复杂，其实使用html5 API把本地的图片转化为base64并不复杂，但是万恶的IE偏偏出来和我们作对，相信前端最想看到的并不是世界和平，而是全世界的IE浏览器都坏掉。

      利用html API把图片转化为base64.

       var render=new FileReader();
      document.getElementById("file").onchange=function(){
                var file=this;
                file=this.files[0];
                render.readAsDataURL(file);
                render.onload=function(){
                    console.log(this.result); //base64数据
                }
            }

      利用ActiveXObject把图片转化为base64


      var b=window.navigator.userAgent,kn;
      var activeXObj;
      var activeXNameList=new Array(“Msxml2.XMLHTTP.6.0”,”Msxml2.XMLHTTP.5.0”,”Microsoft.XMLHTTP”,”Msxml2.XMLHTTP.4.0”,”Msxml2.XMLHTTP.3.0”,”Msxml2.XMLHTTP”,”Microsoft.XMLHTTP”);
                  for(var h=0;h < activeXNameList.length;h++){
                      try{
                         activeXObj=new ActiveXObject(activeXNameList[h]);
                      }catch(e){
                         continue;
                      }
                      if(activeXObj) break;
                  }
                  activeXObj.onreadystatechange=function(){
                      if(activeXObj.readyState < 4) return;
                      var xml_dom=new ActiveXObject(“MSXML2.DOMDocument”);
                      var tmpNode=xml_dom.createElement(“tmpNode”);
                      tmpNode.dataType=”bin.base64”;
                      tmpNode.nodeTypedValue= activeXObj.responseBody;//aa.Read(-1);
                      var base64Data=tmpNode.text.replace(/\n/g,””);
                      var imgData=”data:image/“+kn+”;base64,”+base64Data;
                      console.log(imgData);
                  }
       document.getElementById(“file”).onchange=function(){
                      var arr;
                      arr=this.value.split(“.”);
                      kn=arr[arr.length-1];
                      activeXObj.open(“GET”,this.value,true,””,””);//常报错
                      activeXObj.send(“”);
                  }

      以上两个方法后可以获取到图片的base64数据，html5方法适用IE10以上的浏览器，所以IE9一下的浏览器就必须借助IE专有的ActiveX来获取，但是这种方法在实际操作中经常会报错，无法打开图片文件，这可能是和系统或者浏览器有关系。

      最后来看一下服务器要如何处理base64数据，因为对nodeJs比较熟悉所以这里就以nodeJs为例。

      var express = require(‘express’);
      var router = express.Router();
      var multipart = require(‘connect-multiparty’);//以enctype=”multipart/form-data”上传的数据需要加入这个模块
      var fs = require(“fs”);//加载fs文件处理模块
      var multipartMiddleware = multipart();
      router.post(‘/‘,multipartMiddleware,function(req,res){
                      var b64=req.body.pp1.replace(/^data:image\/\w+;base64,/,””);
                      var dataBuffer=new Buffer(b64,”base64”);//把base64保存在缓存中，在html5中也有这个类
                      fs.writeFile(“public/images/“+req.name,dataBuffer);//这一步把图片保存在public/images/这个目录下
      })
      module.exports = router;
