      javascript闭包浅谈

      Post on 2016-07-02

      javascript闭包是什么，我以前对闭包的概念也是非常的不清楚，闭包是用来干什么的，想必有很多从事前端工作的人对闭包这个东西也有很多疑问吧。

      为了弄清何为闭包我找了很多网站和大牛博客，但由于闭包的概念确实相对较为抽象，而且有些网站或者博客有很多对闭包概念也是描述的不是很清楚，有的甚至只是文字形式的描述了闭包概念并没有结合代码说明。所有在这里我要结合少量代码浅谈一下我对闭包的理解，希望能够帮助到有需要的人，如有不妥的地方希望海涵，并可以指正我的错误。

      以我的理解闭包就是获取函数内部的局部变量的方式。

          var x=0;
          function fn1(){
              var x=1;
          }
          fn1();
          console.log(x);//0

      最后的到的x值毫不意外是0，只是因为fn1后执行但里面的x为fn1的局部变量，这个变量不能传递到函数以外，并且在执行完一次函数之后局部变量x就会被清除。

      var x=0;
      function fn1(){
              var x=1;
              add=function(){
                  x++;
              }
              var fn2=function(){
                  return x;
              }
              console.log(x);//1
              return fn2;
          }
          rn=fn1();
          console.log(rn());//1
          add();
          console.log(rn());//2
          console.log(x);//0

      fn2和add是全局的方法，但是作用域却是在fn1中，fn1中return x;返回一个类fn2，也就是说rn=fn2,当执行rn()时，也就是执行了fn2，返回了fn1函数中的局部变量x；
      执行add(),再执行rn(),这时会得到的2，由此可见函数fn1在执行完一次之后里面的变量并没有在内存中清除，这就是闭包的弊端所在，它不会自己清除变量，如何才能在内存中清掉呢，我想到了一个方法，和fn2
      、add一样写在fn1中del=function(){x=undefined},但是我也不确定这样算不算在内存中清除了这个局部变量，即使这样可以清除变量，但清除局部变量的同时也增加了一个全局的类del，这样看来这种方法也并没有有什么用处。

      有人可能会问，为什么不在fn1()中直接返回局部变量的值呢，这是因为如果返回的是局部变量x的值，即使后面执行了add()后，改变了fn1中的局部变量，rn的值依然只是之前fn1返回的局部变量x，不是现在局部变量x。

      最后真诚的希望各位能够给我提宝贵的意见，互相帮助互相学习。
