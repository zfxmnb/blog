      Canvas 对图片加水印

      Post on 2016-07-21

      对图片加水印是很多网站对自己网站图片进行标识的方法，这样做的原因主要是为了图片的原创性，防止他人盗用图片。

      在此篇文章中将主要对前端使用canvas 来对图片进行加水印方式进行介绍。

       var e=document.createElement(“canvas”);
              function cvs(ele,img,i,r,b){
                  var ctx=e.getContext(“2d”);
                  var image=new Image();
                  image.src=img;
                  var icon=new Image();
                  icon.src=i;
                  var wd=image.width,hg=image.height,rt=(icon.width+(r||10)),bt=(icon.height+(b||10));
                  ele.width=wd;
                  ele.height=hg;
                  icon.onload=function(){
                      ctx.drawImage(image,0,0);console.log((wd-rt)+”,”+(hg-bt));
                      ctx.drawImage(icon,wd-rt,hg-bt);
                      var b64=ele.toDataURL(“image/png”);//把canvas合成的图片转化为base64
                      console.log(b64);
                      return b64;
                  }
              }

      这个方法通过传入底图、水印、水印距右下的距离来合成有水印的图片，canvas则可以是真是存在的元素，也可以是使用createElement创建的canvas。使用canvas方式加水印最后需要把图片转化为base64格式保存或者传到服务器保存，在合成图片的时候要注意不能使用其他源的图片，也就是说不能使用不是同一个域的图片，并不是说在canvas中不能合成不同源的图片，而是由于toDataURL()方法不能把不同源合成图片的base64数据导出来，这可能是出于安全考虑或者其他原因，除了把已有链接的图片作为水印图片的主图，也可以通过html5获取本地图片的base64，传入image.src中。获取本地图片base64方法如下：

              var render=new FileReader();
              document.getElementById(“file”).onchange=function(){
                  var file=this;
                  file=this.files[0];
                  render.readAsDataURL(file);
                  render.onload=function(){
                      console.log(this.result);
                      cvs(e,this.result,i,””,””);
                  }
              }

      html5的的api获取图片base64数据需要IE10以上浏览器才能支持，而canvas需IE9以上浏览器才能支持，也就是说给已有链接的同域图片加水印需要IE9以上浏览器，给本地图片加水印需要IE10以上浏览器。

      在实际的项目中是不会使用这种方式给图片加水印的，而是先把图片在上传到服务器，在服务器上加水印,以nodejs为例后台加水印功能需要nodejs、node-gyp、Python、Cairo、canvas工具及模块支持具体配置参照Windows 下成功安装node-canvas。加水印是通过Python和Cairo思想，引用nodejs的canvas模块，后台实现代码和前台实现代码相似，最后在把base64数据保存成图片，这样就实现了后台加水印功能。
