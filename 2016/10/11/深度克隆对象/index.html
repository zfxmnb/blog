      深度克隆对象

      Post on 2016-10-11

      在前端领域说到克隆大家首先想到的是jquery中的对象克隆clone(),我们通常会使用这个方法把标签对象克隆后再加到页面的某个位置，直接使用$(selector).clone();这种方法的话只能克隆标签的结构和样式，而不能克隆对象的事件方法，当然jquery已经在我们之前想到了这个问题，在jquery中可以使用$(selector).clone('deep');克隆对象，这样克隆的对象的属性方法都会和被克隆对象的属性方法一样，在jquery中我们可以用这个方法完美克隆jquery的对象，那么问题来了，js中的对象如何做到深度克隆，如果不深度克隆会有什么影响。

      在js中的对象主要有标签对象以及JSON数据对象，那么使用深度和不使用深度克隆有什么不一样，接下来就下面例子来分析为什么要深度克隆。

                  var o1={},o2={};
                  o1.x=1;
                  o1.y=function(){return 1};
                  o2=o1;

      console.log(o2.x);// 1
console.log(o2.y());// 1

o2.x=2;
o2.y=function(){return 2};

console.log(o1.x); // 2
console.log(o1.y()); // 2</pre>




      WTF,怎么会有这种事情，没错这种现象确实存在，这就是我们为什么有时候需要深度克隆的原因，至于为什么会这样，我猜可能是因为这样直接赋值的话导致两个对象共用同一个内存空间，无论那个对象的属性方法发生改变都将会影响另外一个对象。

      那么如果需要深度克隆是应该怎么做，之前的o2=o1这种方式是行不通的，o2.x=o1.x o2.y=o1.y这种方式是有用的，如果使一些简单的对象我们可以直接简单的用这种方法实现深度克隆。如果是一个复杂多层的对象要实现深度克隆就需要费一些周章了。

                 function OType(o){
                      if(o= =null) return “Null”;
                      if(o= =undefined) return “Undefined”;
                      return Object.prototype.toString.call(o).slice(8,-1);
                 }
                 function deepClone(obj){
                      var result,oClass=OType(obj);
                      if(oClass = = “Object”){
                          result={};
                      }else if(oClass = = “Array”){
                          result=[];
                      }else{
                          return obj;
                      }
                      for(key in obj){
                          var copy=obj[key];
                          if(OType(copy)==”Object”||OType(copy)= =”Array”){
                              //result[key]=deepClone.call(“”,copy);
                              result[key]=arguments.callee(copy);
                           }else{
                              result[key]=obj[key];
                          }
                      }
                      return result;
                 }

      在这段代码中OType方法目的是为了获取确定对象内部数据的类型，Object.prototype.toString.call(o)取到的是一个字符串格式为[Object ?]，?就是我们要的类型字符串，通常有Object,Function,String,Number等等，在deepClone方法中使用OType的次数会比较多。在deepClone中里面首先判断obj的类型，这样返回的对象一个初始类型，例如：如果是类型为”Object”result={}。再下面就是对对象的循环赋值，这里的循环要进行判断，如果属性对象或者数组的话这需要在次执行deepClone方法一遍，这样才能确保对象深度克隆，如果属性不是对象或者数组的话就无需这个步骤，直接用等号赋值即可，最后返回的result就是克隆的结果。

      在这段代码中有两个关键的地方，一是call的使用，二是arguments的含义及与之配套的callee方法的使用。call的使用在jQuery解析初级篇中有这方面的介绍，想要了解的话可以去看一下；arguments.callee(copy)在这里是调用了自身的方法，这种方式在程序中叫做递归,argument指的就是当前执行的方法，在这里当前方法就是deepClone，而callee与call也是非常的相似，call第一个参数是传给方法中this的，第二个开始才是传参，而callee第一个参数既是传参。
