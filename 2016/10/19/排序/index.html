      排序

      Post on 2016-10-19

      看看日期算下距离天猫的双十一（俗称光棍节实则剁手节-_-）还有二十天了,一大波进击的剁手党严阵以待,在那几天又要发生大规模的流血事件,我闲来无事也喜欢逛下购物网站，当然我多数只是看看而已，我们在浏览购物网站在选择商品的时候通常都会想选择一类商品或一种商品然后再通过不同的排序条件来进行排序，我们常用到的排序有根据价格、销量、评价、综合等这些项目进行排序，排序可以是正序也可以是倒序，其实现和简单就是通过前端发请求到后端，后端把数据处理层特定排序返回前端，这个过程貌似前端没有干啥事情。但是我们可以把这个过程放到前端来分析，如何在前端把商品对象以想要的排序方式进行排序，排序无非就是从小到大或者从大到小的排序，一般的数组的排序我们已经驾轻就熟了，如果只是单纯的排序不用对数据进行增删的操作的或可以直接用sort()进行排序，而如果在排序中需要对数据进行修改也可以通过多重循环来获得排序。
      而对于数组值为对象的数组又如何对其进行排序呢，其实排序方式都是一样的，记下来就来罗列一下我觉得可行的几种排序方法。

      function Sort1(a,attr,inverted){
                      var ar=[a[0]],l=a.length;
                      for(var i=1;i < l;i++){
                          var ai=a[i][attr];
                          for(var j=i-1;j > -1;j–){
                              var aj=ar[j][attr];
                              if((inverted||false)&&(ai < ar[i-1][attr]|| ai = = aj)){
                                  ar.splice(ar.length,0,a[i]);
                                  break;
                              }else if(!(inverted||false)&&(ai > ar[i-1][attr]||ai = =aj)){
                                  ar.splice(ar.length,0,a[i])
                                  break;
                              }
                             if(j > 0){
                                  var a_j = ar[j-1][attr];
                                  if((inverted||false)&&(ai > aj&&ai < a_j||ai= =a_j)){
                                          ar.splice(j,0,a[i])
                                          break;
                                  }else if(!(inverted||false)&&(ai < aj&&ai > a_j||ai= =a_j)){
                                          ar.splice(j,0,a[i])
                                          break;
                                  }
                             }else{
                                  ar.splice(0,0,a[i]);
                             }
                          }
                      }
                      return ar;
              }

      function Sort2(a,attr,inverted){
    var ar=[],l=a.length;
    for(var i=0;i < l;i++){
        var mArr=a[0],index=0;
        for(var j=0;j < a.length;j++){
            if(mArr[attr] < a[j][attr]||mArr[attr]==a[j][attr]){
                mArr=a[j];
                index=j;
            }
        }
        a.splice(index,1);
        (inverted||false)?ar.push(mArr):ar.unshift(mArr)
    }
    return ar
}

function Sort3(a,attr,inverted){
        function sort(a,b){
            return a[attr] > b[attr]?1:-1
        }
        function sortInv(a,b){
            return b[attr] > a[attr]?1:-1
        }
        return a.sort((inverted||false)?sortInv:sort);
}


      以上的三种方法都实现了对数组arr的正反两种排序，这三种方法实现的方式不一样，首先来说下sort1，这个方法通过两个循环用splice方法把原数组的对象插入到新数组中去，这种方法对原来的数组没有什么影响，结果返回的是一个新的数组,但是这个方法看到这几个辣么长的判断就够让人头疼的了；第二种排序方法sort2就是我们比较能想到的方法，这个方法通过循环取最大属性值的对像，在通过push或者unshift的方法建立新的数组，这种方法的弊端就是会把原来的数组arr全部删光，至于明明是arr传值给sort2，为什么会改变arr的值，这个问题可以参考《深度克隆》为什么要深度克隆部分。最后一个排序方法是sort3
      ，这个方法只有短短的几行代码，这个方法用到了js原生的sort方法，只要用过这个方法的人应该都能比较好理解。单从代码的的方面考虑量的方面考虑的话排序应该是sort3>sort2>sprt1,但是为了保险起见，我还是对这几个方法进行了性能测试，我使用一个长度为20的对象数组，数组结构是这样的arr = [{"name":"","age":}，然后我用循环执行每个方法1e5次，记录他们执行1e5次所需要的时间，得出结果最快的是sort3大概150ms，其次是sort1大概750ms，最慢的是sort2大概850ms，这些数据只是在特定情况下得到的，有可能换一个电脑或者换一个浏览器，结果又会有一下偏差，但是这些数据大概可以对比出这三种方法性能上的排序，可以给以后做排序是选择方法时做参考。
